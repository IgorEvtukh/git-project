# Документация архитектуры базы данных (PostgreSQL)

Вся логика платформы (пользователи, матчи, исходы и ставки) опирается на реляционную структуру PostgreSQL. Ниже описаны ключевые связи между таблицами из `schema.sql`.

## Связи между таблицами

### 1. `matches` (Матчи) -> `markets` (Маркеты)
* **Тип связи:** Один матч может содержать много маркетов (например: Победитель, Тотал, Фора), но конкретный маркет всегда принадлежит только одному матчу.
* **Связь:** `Один-ко-многим` через `markets.match_id -> matches.id`.

### 2. `users` (Пользователи) -> `bets` (Ставки)
* **Тип связи:** Один пользователь может сделать сколько угодно ставок, но у каждой конкретной ставки есть только один владелец.
* **Связь:** `Один-ко-многим` через `bets.user_id -> users.id`.

### 3. `outcomes` (Исходы) -> `bets` (Ставки)
* **Тип связи:** На один и тот же исход (например, «Победа Спартака») могут поставить тысячи разных пользователей (будет много записей в таблице `bets`), но сама одиночная ставка привязана строго к одному исходу.
* **Связь:** `Один-ко-многим` через `bets.outcome_id -> outcomes.id`
