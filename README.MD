# Betting Platform API Reference & Infrastructure

Добро пожаловать в техническую документацию букмекерческой платформы. Данный репозиторий содержит полное описание архитектуры, спецификации API (v1) и конфигурацию инфраструктуры для развертывания сервиса.

## 🏛 Архитектура системы

Проект спроектирован с учетом высоких нагрузок (Highload) и требований к финансовой безопасности:
* **БД (PostgreSQL):** Хранение данных пользователей, балансов, матчей и истории ставок. Оптимизировано индексами.
* **Кэширование (Redis):** Снижение нагрузки на основную БД при чтении лайв-линии. TTL кэша — 60 секунд.
* **Безопасность:** Авторизация на базе JWT-токенов. Защита от повторных списаний через `Idempotency-Key`.

## 📁 Структура документации

* `schema.sql` — Спецификация структуры базы данных и поисковых индексов.
* `redis_cache.md` — Стратегия кэширования и инвалидации данных в Redis.
* `CONTRIBUTING.md` — Стайлгайд и правила оформления документации для разработчиков.
* `api/v1/` — Справочник эндпоинтов API:
  * `auth/` — Регистрация, авторизация (JWT) и профиль пользователя.
  * `matches/` — Получение линии матчей и SSE-стриминг коэффициетнов.
  * `bets/` — Прием ставок и проверка баланса.

## 🚀 Быстрый старт (Локальное развертывание)

Для запуска окружения (базы данных PostgreSQL и кэша Redis) в проекте используется Docker. Это позволяет развернуть все сервисы одной командой без необходимости устанавливать их на компьютер по отдельности.

### Предварительные требования
* Установленный Docker Desktop
* Установленный Git

### Запуск окружения

```bash
# 1. Склонируйте репозиторий и перейдите в корневую папку:
git clone <ссылка_на_репозиторий>
cd <название_папки_проекта>

# 2. Запустите контейнеры базы данных и кэша в фоновом режиме:
docker-compose up -d

# 3. Проверить статус запущенных контейнеров:
docker ps
