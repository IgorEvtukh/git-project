# Руководство по разработке (Contributing Guide)

Привет! Это руководство описывает стандарты оформления API-документации в нашем проекте.

## 1. Правила оформления Markdown

* **Формат файлов:** Каждый HTTP-метод описывается в отдельном файле с расширением `.md` (например, `get_balance.md`, `post_deposit.md`).
* **Названия параметров:** Все параметры запросов и ответов (ключи в JSON) всегда пишутся в нижнем регистре и в стиле `snake_case` (например, `transaction_id`).
* **Даты и время:** Для всех временных меток строго используется формат ISO 8601 в UTC (например, `2026-06-07T15:00:00Z`).

## 2. Стандарты Git и коммитов

Мы используем соглашение **Conventional Commits** для оформления сообщений. Название коммита должно начинаться с префикса:

* `docs:` — для изменений в документации (основной тип для этого проекта).
* `fix:` — если исправлена ошибка или опечатка в синтаксисе.
* `chore:` — рутинные задачи (обновление `.gitignore`, настройка окружения).
