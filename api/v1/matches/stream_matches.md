# Стриминг обновлений линии в реальном времени (SSE)

Метод позволяет фронтенду открыть постоянное соединение для получения мгновенных обновлений коэффициентов и статусов матчей без повторных HTTP-запросов.

* **URL метода:** `/api/v1/matches/stream`
* **HTTP-метод:** `GET`
* **Заголовки (Headers):** `Accept: text/event-stream` (строго обязательно для SSE)
* **Авторизация:** Не требуется

### Протокол взаимодействия

После успешного подключения сервер удерживает соединение открытым и отправляет текстовые события (события генерируются только тогда, когда в матче что-то изменилось: поменялся коэффициент, забили гол или заблокировали маркет).

### Формат отправляемых данных (Event Payload)

Каждое событие приходит в формате JSON, содержащем только изменившиеся данные, привязанные к конкретному `outcome_id` или `match_id`.

#### Пример 1: Изменение коэффициента в лайве (`event: coefficient_update`)
```json
{
  "match_id": 85732,
  "market_id": "main_outcome",
  "outcome_id": 1001,
  "coefficient": 1.95
}
```

#### Пример 2: Блокировка приема ставок на маркет (`event: market_lock`)
```json
{
  "match_id": 85732,
  "market_id": "total_goals_2.5",
  "status": "locked"
}
```
