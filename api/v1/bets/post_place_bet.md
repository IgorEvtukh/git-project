# Создание новой ставки (Прием купона)

Метод регистрирует новую ставку пользователя в системе, проверяет доступность баланса и резервирует средства под купон.

* URL метода: /api/v1/bets/place
* HTTP-метод: POST
* Авторизация: Требуется (Токен пользователя)

### Тело запроса (Request Body)

Приложение отправляет на сервер JSON-пакет со следующими параметрами ставки:

```json
{
  "match_id": 85732,
  "outcome_id": "home_win",
  "coefficient": 2.10,
  "bet_amount": 1000.00
}
```

### Описание параметров запроса

| Параметр | Тип данных | Обязательный | Описание |
| :--- | :--- | :--- | :--- |
| match_id | Integer | Да | Уникальный идентификатор матча в системе |
| outcome_id | Integer | Да | Код исхода (например: home_win, away_win, draw) |
| coefficient | Float | Да | Коэффициент ставки на момент отправки купона |
| bet_amount | Float | Да | Сумма ставки в валюте пользователя |

### Успешный ответ (Response 200 OK)

Если ставка успешно зарегистрирована, сервер возвращает статус успеха, ID созданной ставки и обновленный баланс пользователя:

```json
{
  "status": "success",
  "data": {
    "bet_id": 948271,
    "balance": {
      "balance_available": 500.00,
      "balance_bonus": 500.00,
      "balance_total": 1000.00
    }
  }
}
```
### Описание параметров успешного ответа (внутри объекта data)

| Параметр | Тип данных | Обязательный | Описание |
| :--- | :--- | :--- | :--- |
| bet_id | Integer | Да | Уникальный идентификатор созданной ставки в системе |
| balance | Object | Да | Объект с обновленным состоянием счетов пользователя |

Или если за время оформления купона коэффициент изменился:

```json
{
  "status": "error",
  "error_code": "COEFFICIENT_CHANGED",
  "message": "Коэффициент изменился. Пожалуйста, обновите купон"
}
```

### Ответ сервера при ошибке (Response 400 / 422)

Если ставку принять невозможно, сервер возвращает статус ошибки и JSON-пакет с описанием причины:

```json
{
  "status": "error",
  "error_code": "INSUFFICIENT_FUNDS",
  "message": "Недостаточно средств на балансе пользователя"
}
```

### Описание параметров ответа при ошибке

| Параметр | Тип данных | Обязательный | Описание |
| :--- | :--- | :--- | :--- |
| status | String | Да | Всегда error для неуспешных запросов |
| error_code | String | Да | Строковый технический код ошибки для фронтенда (COEFFICIENT_CHANGED, INSUFFICIENT_FUNDS) |
| message | String | Да | Человекочитаемое описание ошибки для отображения пользователю |
