# Пополнение баланса пользователя

Метод создаёт запрос на пополнение игрового счета через платёжный шлюз.

* URL метода: /api/v1/user/deposit
* HTTP-метод: POST
* Авторизация: Требуется (Token)

### Тело запроса (Request Body JSON)

В теле запроса передаются параметры для инициализации платежа:

| Параметр | Тип данных | Обязательный | Описание |
| :--- | :--- | :--- | :--- |
| amount | Float | Да | Сумма пополнения (например, 500.00) |
| currency | String | Да | Трехзначный код валюты (RUB, USD, THB) |
| payment_method | String | Да | Идентификатор платёжной системы (card, sbp, crypto) |

### Успешный ответ (Response 200 OK)

Сервер регистрирует транзакцию в нашей системе и возвращает ссылку для перехода на платежную страницу эквайринга:

```json
{
  "status": "success",
  "data": {
    "transaction_id": "tx_987654321",
    "amount": 500.00,
    "currency": "RUB",
    "status": "pending",
    "payment_url": "[https://payment-gateway.com/pay/invoice_abc123xyz](https://payment-gateway.com/pay/invoice_abc123xyz)",
    "created_at": "2026-06-04T15:49:00Z"
  }
}
```

### Описание параметров успешного ответа

| Параметр | Тип данных | Обязательный | Описание |
| :--- | :--- | :--- | :--- |
| transaction_id | String | Да | Уникальный идентификатор транзакции в нашей системе |
| amount | Float | Да | Итоговая сумма платежа, которая будет зачислена |
| currency | String | Да | Трехзначный код валюты (например, RUB) |
| status | String | Да | Текущий статус транзакции. Возможные значения (Enum):<br>• pending — ожидание оплаты (дефолт для этого метода)<br>• processing — платеж в обработке шлюзом<br>• success — деньги успешно зачислены<br>• fail — ошибка проведения платежа<br>• expired — время действия ссылки истекло |
| payment_url | String | Да | Прямая ссылка на платёжный шлюз для ввода реквизитов или отображения QR-кода |
| created_at | String (ISO 8601) | Да | Дата и время создания инвойса в формате UTC |
