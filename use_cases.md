# Сценарии использования системы (Use Cases)

## UC-1: Размещение одиночной ставки авторизованным пользователем

Актёр: Авторизованный клиент платформы
Предусловие: Пользователь вошел в систему, на его балансе есть средства, выбран исход матча.

### Основной успешный сценарий (Happy Path):

1. Клиент в мобильном приложении нажимает кнопку «Сделать ставку».
2. Мобильное приложение отправляет POST-запрос на эндпоинт /api/v1/bets/.
3. Наш API-сервер принимает запрос, проверяет JWT-токен пользователя и защитный заголовок Idempotency-Key.
4. Система делает быстрый запрос в Кэш Redis, чтобы мгновенно проверить, не изменился ли коэффициент на этот матч за последние секунды (ведь нам нельзя принимать ставку по старому кэфу).
5. Система проверяет баланс пользователя и фиксирует списание денег и создание купона ставки. Так как это финансовая операция и нужна 100% надежность, эти данные записываются строго в БД Postgre SQL.
6. API-сервер возвращает мобильному приложению ответ 200 OK с деталями успешно принятой ставки.
7. Пользователь видит на экране уведомление: «Ставка принята!».
